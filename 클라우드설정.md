# 저장된 일정 — 클라우드(공유) 설정 가이드 · Firebase

저장본을 **어느 기기에서나 같은 목록으로 보고, 동행자와 공유**하기 위해 Firebase Firestore를 사용합니다.
이 앱은 빌드 과정이 없어서, Firebase SDK 없이 **Firestore REST API를 직접 호출**합니다.
키가 비어 있으면 자동으로 **이 브라우저 로컬 저장**으로 동작하므로, 설정 없이도 앱은 그대로 쓸 수 있습니다.

`app.js`에는 이미 보내주신 `firebaseConfig`가 들어가 있습니다. 남은 건 **Firestore를 켜고 접근 규칙을 정하는 것**뿐입니다.

## 1. Firestore 데이터베이스 만들기

1. https://console.firebase.google.com → 프로젝트 **trip-plan-4b079** 선택.
2. 좌측 메뉴 **빌드(Build) → Firestore Database** → **데이터베이스 만들기**.
3. 위치(region)는 가까운 곳(예: asia-northeast3 서울) 선택 → 만들기.

## 2. 보안 규칙 설정 (중요)

이 앱은 로그인 없이 API 키만으로 접근하므로, 규칙에서 접근을 허용해야 합니다.
**Firestore Database → 규칙(Rules)** 탭에 아래를 붙여넣고 **게시(Publish)**:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /saved_trips/{doc} {
      allow read, write: if true;
    }
  }
}
```

> ⚠️ 위 규칙은 **키를 아는 사람이면 누구나** `saved_trips`를 읽고 쓸 수 있다는 뜻입니다.
> (Firebase 웹 `apiKey`는 비밀이 아니며, 접근 제어는 이 규칙이 담당합니다.)
> 가족·동행자끼리 쓰는 여행 일정 정도면 충분하지만, 민감한 정보는 넣지 마세요.
> 나중에 로그인 기반으로 잠그고 싶으면 알려주세요.

## 3. 끝 — 바로 사용

설정 끝입니다. 페이지를 새로고침하면:

- **현재 일정 저장** → 지금 화면(일정·지출·골프·메모·기간·환율)을 이름 붙여 클라우드에 저장.
- **저장된 일정 불러오기** → 목록에서 **불러오기 · 이름변경 · 삭제**. 항목 옆에 **클라우드** 표시.
- 다른 PC·휴대폰에서 같은 페이지를 열어도 같은 목록이 보입니다.

## 동작 정리

- 저장본은 Firestore의 `saved_trips` 컬렉션에 문서 하나로 저장됩니다.
  (필드: `name`, `saved_at`, `data` — `data`는 화면 상태 전체를 담은 맵)
- `app.js`의 `firebaseConfig`를 비우면 같은 기능이 **이 브라우저 로컬**로만 동작합니다.
- 인터넷이 끊기거나 규칙이 막혀 있으면 목록에 "불러오지 못했습니다" 안내가 표시됩니다.
