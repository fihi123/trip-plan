# 저장된 일정 클라우드 공유 설정

저장본을 여러 기기에서 공유하려면 Firebase Firestore를 사용합니다. 앱은 기본적으로 **이 브라우저 로컬 저장**으로 동작하고, 사용자가 **클라우드 공유 코드**를 설정한 경우에만 클라우드 저장본을 읽고 씁니다.

공유 코드는 브라우저에만 저장되며, 앱은 공유 코드를 SHA-256으로 해시한 값 아래에 저장본을 만듭니다. 같은 코드를 입력한 사람만 같은 저장본 목록을 보게 됩니다.

## 1. Firestore 데이터베이스 만들기

1. https://console.firebase.google.com 에서 프로젝트 `trip-plan-4b079` 선택.
2. 좌측 메뉴 **빌드 → Firestore Database**.
3. 데이터베이스 만들기.
4. 위치는 가까운 리전 선택.

## 2. 보안 규칙 설정

Firestore Database → 규칙 탭에 아래 규칙을 게시합니다.

```txt
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /trip_groups/{group}/saved_trips/{doc} {
      allow read, write: if group.matches('^[a-f0-9]{32}$');
    }
  }
}
```

이전 문서의 `saved_trips` 전체 공개 규칙을 쓰고 있다면 삭제하세요. 새 앱은 `trip_groups/{공유코드해시}/saved_trips` 경로를 사용합니다.

## 3. 앱에서 사용

1. 앱에서 **저장된 일정 불러오기**를 엽니다.
2. **클라우드 코드 설정**을 누릅니다.
3. 동행자와 공유할 6자 이상의 코드를 입력합니다.
4. 같은 코드를 입력한 기기끼리 저장본 목록을 공유합니다.

코드를 설정하지 않으면 저장본은 이 브라우저에만 저장됩니다. **로컬로 전환**을 누르면 클라우드 연결만 해제되고 클라우드 데이터는 삭제되지 않습니다.

## 주의

- Firebase 웹 `apiKey`는 비밀값이 아닙니다. 접근 범위는 Firestore 규칙과 공유 코드 경로로 제한합니다.
- 공유 코드는 짧거나 흔한 단어보다 길고 추측하기 어렵게 만드세요.
- 민감한 여권번호, 카드번호 같은 정보는 저장하지 마세요.
